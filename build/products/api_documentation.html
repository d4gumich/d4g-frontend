Hangul API Documentation

Overview

This documentation covers the usage of two API endpoints of Hangul provided by https://d4gumsi.pythonanywhere.com for handling document processing and summarization. Below is an outline of the endpoints, payload structure, and the sequence of API calls.

Endpoints

1. Document Processing Endpoint

URL: https://d4gumsi.pythonanywhere.com/api/v2/products/hangul

Method: POST

Description: This endpoint processes a given PDF document based on the specified payload parameters.

File: A PDF file to be sent along with the payload.

Response: The response will be a dictionary which contains various keys, one of which is document_summary_parameters that will be used for the next API call.

Payload Parameters

kw_num (str): Number of keywords to extract.

Return_ALL (bool): A flag to indicate if all available data should be returned. If this one True, it overrides all the rest.

Document-related flags (bool): Flags to specify if certain document attributes should be extracted:

  • document_language: The language of the document.
  • document_title: A selection of the most likely titles of the document with the weight of each title likelihood.
  • document_summary: The summary of the document and the keywords of the summary.
  • content: Content to be displayed, the text of up to 4 pages of the document.
  • report_type: Type of report.
  • locations: Locations mentioned in the report.
  • full_content: Clean text of the pdf.
  • markdown_text: Markdown of the pdf text.
  • document_theme: Theme of the document.
  • new_detected_disasters: Disasters detected in the document.

pdf Metadata information: This information is extracted from the metadata of the pdf file. It is available in the metadata dictionary of the response (All are bool type).

  • author: Author of the pdf document.
  • doc_created_date: The date of the document creation.
  • doc_modified_date: The date of the document modification.
  • num_of_pages: Number of pages in the document.
  • acharsPerPage: Characters per page in the document.

Code example:

# Define API URL api_url_1 = 'https://d4gumsi.pythonanywhere.com/api/v2/products/hangul' # Define Payload payload = { 'kw_num': "5", 'Return_ALL': False, 'document_language': False, 'document_title': True, 'document_summary': False, 'content': False, 'report_type': True, 'locations': True, 'full_content': True, 'markdown_text': True, 'document_theme': False, 'new_detected_disasters': True, "Author": False, "doc_created_date": False, "doc_modified_date": False, "num_of_pages": False, "charsPerPage": False, } # Define File to be uploaded files = { "file": ("filename.pdf", open("filepath/filename.pdf", "rb")), } # Make the Request response_1 = requests.post(api_url_1, files=files, data=payload)

The response keys are the following:

  • 'metadata': Returns a dictionary with metadata.
  • 'document_language': Returns the language and the language score.
  • 'document_title': List of title â€“ Likelihood pairs
  • 'document_summary_parameters': Dictionary with inputs to generate the summary.
  • 'content': List of strings with the text of the first 4 pages of the document.
  • 'report_type': string with the report-type.
  • 'locations': Dictionary with locations and number of occurrences.
  • 'full_content': Clean string with all the text of the document.
  • 'markdown_text': String with the text of the document in markdown type.
  • 'document_theme': Themes of the document in a list.
  • 'new_detected_disasters': List of detected disasters in the document.

2. Document Summary Endpoint

URL: https://d4gumsi.pythonanywhere.com/api/v2/products/summary

Method: POST

Description: This endpoint generates a summary of the document based on the parameters received from the first API call and also extracts the keywords of the summary.

Payload Parameters: The parameters obtained from the first API call (document_summary_parameters ). Pass the exact dictionary that was generated in the first API response with the key document_summary_parameters.

Code example:

# Define API URL api_url_summary = "https://d4gumsi.pythonanywhere.com/api/v2/products/summary" # Extract summary parameters from the first response summary_parameters = response_1.json()["document_summary_parameters"] # Check required flag and make the second request if (payload["document_summary"]) or (payload["Return_ALL"]): response_summary = requests.post(api_url_summary, json=summary_parameters) # Print the generated summary print("GENERATED SUMMARY: ", response_summary.json()["summary"])

Complete Workflow

  1. Prepare Payload and File
    • Set the necessary payload parameters.
    • Provide the PDF file path and name.

  2. Make the First API Call
    • Send the PDF file and payload to the Document Processing Endpoint.
    • Retrieve `document_summary_parameters` from the response.

  3. Conditionally Make the Second API Call
    • If `document_summary` or `Return_ALL` is set to `True`, use the obtained parameters to request a document summary.

import requests import time # Define the URLs api_url_1 = 'https://d4gumsi.pythonanywhere.com/api/v2/products/hangul' api_url_summary = "https://d4gumsi.pythonanywhere.com/api/v2/products/summary" filename = "" # ADD here the file name filepath = "" # ADD here the file path pdf_path = f"{filepath}/{filename}.pdf" # Create the file and kw that will be passed to the API payload = { "kw_num": "5", "Return_ALL": False, 'document_language': False, 'document_title': True, 'document_summary': False, 'content': False, 'report_type': True, 'locations': True, 'full_content': True, 'markdown_text': True, 'document_theme': False, 'new_detected_disasters': True, "Author": False, "doc_created_date": False, "doc_modified_date": False, "num_of_pages": False, "charsPerPage": False, } # This is the file to be sent to the API files = { "file": (f"{filename}.pdf", open(pdf_path, "rb")), } print("----------RUNING the API CALLS----------") # Try to get a response from the API # If it does not work, try a second time # To allow the server to restart try: response_1 = requests.post(api_url_1, files=files, data=payload) except: time.sleep(2) response_1 = requests.post(api_url_1, files=files, data=payload) # Get the data from the first API call # to use the parameters for the second API call summary_parameters = response_1.json()["document_summary_parameters"] print("Response keys: ", response_1.json().keys()) time.sleep(1) try: if (payload["document_summary"]) | (payload["Return_ALL"]): response_summary = requests.post(api_url_summary, json=summary_parameters) print("GENERATED SUMMARY: ",response_summary.json()["summary"]) # If no response, try again except: time.sleep(2) if (payload["document_summary"]) | (payload["Return_ALL"]): response_summary = requests.post(api_url_summary, json=summary_parameters) print("GENERATED SUMMARY: ",response_summary.json()["summary"])
